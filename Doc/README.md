# 篮球比赛记分牌 — 项目说明

> 基于 STM32F103C8T6 的篮球比赛电子记分牌：OLED 实时显示两队比分与比赛时间，4 键独立控制计分，蜂鸣器按键提示音，支持多小节倒计时与胜负判定。

---

## 1. 项目功能

- **两队计分**: 甲方 (Team A) 与乙方 (Team B) 独立计分，上限 199 分。
- **分数类型**: 支持 +1 分 (罚球)、+2 分 (投篮)、+3 分 (三分球)。
  - 短按 KEY2/KEY4: +2 分
  - 长按 KEY2/KEY4 (>0.6s): +3 分
- **按键提示音**: 每次计分成功时蜂鸣器发出短促提示音。
- **比赛时间倒计时**: 每节 12 分钟 (可配置)，共 4 节，TIM2 硬件定时器精确计时。
- **小节自动切换**: 每节时间到自动进入暂停，按键后开始下一节。
- **比赛结束判定**: 第 4 节时间到或手动结束，OLED 显示最终比分与胜负结果。
- **手动控制**: 同时按下 KEY1+KEY3 可提前结束比赛；比赛结束后按 KEY1+KEY3 重新开始。

## 2. 硬件连接

| 模块 | 引脚 | 说明 |
|------|------|------|
| OLED SCL | PB6 | 软件 I2C 时钟 |
| OLED SDA | PB7 | 软件 I2C 数据 |
| 蜂鸣器 (无源) | PB0 | TIM3_CH3 PWM 输出，按键提示音 |
| KEY1 | PA15 | 甲方 +1 分 |
| KEY2 | PA14 | 甲方 +2 分 (长按 +3 分) |
| KEY3 | PA13 | 乙方 +1 分 |
| KEY4 | PA12 | 乙方 +2 分 (长按 +3 分) |
| 模式选择 | PC13 | 高电平=LED 模式 (必须拉高，关闭板载数码管) |

> **注意**: PA12/PA13/PA14/PA15 需要关闭 JTAG/SWD 复用才能作为普通 GPIO 按键使用。

## 4. 按键说明

| 按键 | 操作 | 功能 |
|------|------|------|
| KEY1 (PA15) | 短按 | 甲方 +1 分 (罚球) |
| KEY2 (PA14) | 短按 / 长按 (>0.6s) | 甲方 +2 分 (投篮) / +3 分 (三分) |
| KEY3 (PA13) | 短按 | 乙方 +1 分 (罚球) |
| KEY4 (PA12) | 短按 / 长按 (>0.6s) | 乙方 +2 分 (投篮) / +3 分 (三分) |

## 5. 软件设计详解

### 5.1 系统状态机

```
                    ┌──────────┐
          上电/复位 →│   IDLE   │←──────────────┐
                    └────┬─────┘                │
                 按任意键│                       │
                         ↓                      │
                    ┌──────────┐   KEY1+KEY3    │
                    │ PLAYING  │──────────────→ │
                    └────┬─────┘                │
              本节时间到│    ↑ 按任意键          │
                         ↓    │                  │
                    ┌──────────┐                │
                    │  PAUSED  │                │
                    └────┬─────┘                │
              第4节到│                            │
                     ↓                            │
                ┌──────────┐  KEY1+KEY3          │
                │ GAMEOVER │─────────────────────┘
                └──────────┘
```

### 5.2 比赛计时器 (TIM2)

| 配置项 | 值 | 说明 |
|--------|----|------|
| 时钟源 | APB1 = 72 MHz | |
| PSC | 72-1 = 71 | 72 分频 → 1 MHz |
| ARR | 1000-1 = 999 | 1000 分频 → 1 KHz (1ms 中断) |
| 中断 | Update 事件 | 每 1ms 触发一次 |

在 ISR 中:
1. `g_sys_tick++` — 全局毫秒计数器 (用于按键长按判断)
2. 每 1000 次 (1 秒) 比赛时间减一秒
3. 时间到自动切换小节或结束比赛

### 5.3 长按检测机制

KEY2 或 KEY4 按下后，主循环通过 `Key_IsPressed()` 持续检测按键状态:
- 600ms 内松手 → 短按 → +2 分
- 持续按住超过 600ms → 长按 → +3 分

这 600ms 的轮询过程中 TIM2 中断仍然正常运行，比赛计时不受影响。

### 5.4 OLED 显示布局

**比赛画面 (4 行 × 16 字符)**:
```
A:065  Q1  12:00     ← 甲方分数 + 小节 + 倒计时
B:058                 ← 乙方分数
+1 +2(+3长按)         ← 操作提示 (播放中) / 状态 (暂停时)
1/2:A  3/4:B          ← 按键对应队伍
```

**比赛结束画面**:
```
===GAME OVER===
A:065  B:058
Winner: Team A       (或 Team B / Draw!)
1+3:NewGame          ← 按 KEY1+KEY3 重新开始
```

## 6. 关键设计点

### 6.1 为什么用 TIM2 中断而不是 Delay 计时?
- **精度高**: 72MHz 晶振 → 1MHz 计数 → 1ms 中断误差 <0.001%
- **非阻塞**: 比赛计时与按键处理、显示刷新并行进行
- **长按检测依赖**: `g_sys_tick` 为长按提供精确的时间基准

### 6.2 为什么短按 +2、长按 +3?
- 4 个物理按键不足以给两个队伍各分配 3 个分数键 (需要 6 键)
- 长按方案复用现有按键，无需额外硬件
- 600ms 的阈值远小于人手反应时间 (~200ms)，长按操作直观易用

### 6.3 小节切换为什么需要按键确认?
- 实际比赛中，小节间有休息时间
- 按键确认确保裁判/操作员准备好后计时才开始
- 自动切换会打乱比赛节奏

### 6.4 分数上限 199 的原因
- OLED 每行 16 字符，3 位数字可完整显示
- 篮球比赛单队得分极少超过 199 分
- 防止 uint8_t 溢出 (255 上限)

## 7. 可扩展功能

1. **球员得分统计**: 在比赛暂停/小节间隙切换到球员统计子界面，记录每队 5 名球员的个人得分。
2. **24 秒进攻计时**: 使用 TIM4 增加独立的 24 秒进攻计时器。
3. **犯规/暂停计数**: 显示每队犯规次数与剩余暂停次数。
4. **外部大屏驱动**: 通过 UART/SPI 输出数据到 LED 点阵大屏幕。
5. **节间/终场长鸣**: 本节/全场结束时蜂鸣器发出 2 秒长鸣。
