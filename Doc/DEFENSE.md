## 一、项目背景与需求分析

### 1.1 项目背景

在各类篮球赛事中，准确的实时计分与计时是比赛公正性的基础保障。传统手动翻牌记分牌存在操作繁琐、缺乏提示音、信息单一等问题。本项目设计一款基于 STM32F103C8T6 的篮球比赛电子记分牌，使用 OLED 屏幕实时显示两队比分与比赛时间，通过 4 个物理按键控制计分操作，蜂鸣器提供按键提示音，TIM2 硬件定时器实现精确的多小节倒计时。

### 1.2 系统需求分析

| 需求类别 | 需求描述 | 实现方式 |
|----------|----------|----------|
| 基本功能-计分 | 记录和显示两个队伍的得分 | 甲方(Team A) / 乙方(Team B) 独立变量存储，OLED 显示 |
| 基本功能-按键 | 4 个按键控制加 1、2、3 分 | KEY1/3: +1 分；KEY2/4: 短按 +2 分，长按 +3 分 |
| 基本功能-音效 | 每次计分成功有提示音 | 无源蜂鸣器，TIM3 PWM 驱动，鸣叫 50ms |
| 扩展功能-计时 | 记录比赛时间倒计时 | TIM2 1ms 中断，用户可选 5s / 5min / 15min / 30min 每节 |
| 扩展功能-小节 | 记录比赛小节数 | 4 节，每节时间到自动暂停，按任意键进入下一节 |
| 扩展功能-结束 | 比赛结束时显示结果 | OLED 清屏显示 GAME OVER + 胜者，任意键回到选时间界面 |

## 二、系统总体设计

### 2.1 硬件架构

```
┌──────────────────────────────────────────────────────┐
│                  STM32F103C8T6                        │
│                                                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │  KEY1~4  │  │  Buzzer  │  │   OLED   │        │
│  │ PA12~15  │  │ PB0 TIM3 │  │ PB6/7 I2C│        │
│  │ 上拉输入   │  │ CH3 PWM  │  │ 软件 I2C  │        │
│  └──────────┘  └──────────┘  └──────────┘        │
│                                                       │
│  ┌──────────────────────────────────────┐          │
│  │  TIM2 (定时器)                         │          │
│  │  PSC=72-1, ARR=1000-1 → 1ms 中断    │          │
│  │  比赛倒计时 + 全局毫秒计数             │          │
│  └──────────────────────────────────────┘          │
└──────────────────────────────────────────────────────┘
```

### 2.2 软件架构

系统采用**前后台架构** (main 循环 + TIM2 定时器中断):

- **前台 (main 循环)**: OLED 区域刷新、按键扫描与消抖、计分逻辑处理、状态机调度
- **后台 (TIM2 中断)**: 1ms 全局计数、比赛时间倒计时、小节切换、比赛结束判定

### 2.3 系统状态机

```
IDLE ──(按 KEY1~4 选择时间)──→ PLAYING ──(每节时间到)──→ PAUSED
                                                           │
                              ──(任意键)────────────────────┘
                                                           │
                              ──(第4节结束)─────────────────
                                                           ↓
                                                        GAMEOVER
                                                           │
                              ──(任意键)────────────────────┘
                                                           ↓
                                                        IDLE
```

### 2.4 OLED 显示布局（4 行 × 16 字符）

**时间选择界面 (IDLE)**

| 行 | 内容 |
|----|------|
| 1 | `Select Time:    ` |
| 2 | `K1:5s   K2:5m   ` |
| 3 | `K3:15m  K4:30m  ` |
| 4 | `Press K1-K4     ` |

**比赛界面 (PLAYING / PAUSED)**

| 行 | 内容 |
|----|------|
| 1 | `A: XXX          ` (甲方分数) |
| 2 | `B: XXX          ` (乙方分数) |
| 3 | `Q1 12:00        ` (节数 + 倒计时) |
| 4 | `1/3:+1 2/4:+2 L3` (比赛中) 或 `PAUSE-ANY KEY  ` (暂停) |

**比赛结束界面 (GAMEOVER)**

| 行 | 内容 |
|----|------|
| 1 | `===GAME OVER===` |
| 2 | `Winner: Team A  ` / `Winner: Team B  ` / `Draw!           ` |
| 3 | `A: XXX  B: XXX  ` |
| 4 | `ANY KEY=RESTART ` |

## 三、硬件设计

### 3.1 主控芯片

选用 STM32F103C8T6:
- Cortex-M3 内核，主频 72MHz
- 64KB Flash，20KB SRAM
- 3 个通用定时器 (TIM2/3/4)
- 丰富的 GPIO 端口

### 3.2 OLED 显示模块

- 型号: 0.96 寸 SSD1306，分辨率 128×64
- 接口: 软件模拟 I2C (PB6 SCL / PB7 SDA)
- 字体: 8×16 ASCII 字模，4 行 × 16 列

### 3.3 按键电路

| 按键 | GPIO | 功能（IDLE状态） | 功能（PLAYING状态） |
|------|------|-----------------|-------------------|
| KEY1 | PA15 | 选择 5 秒演示 | 甲方 +1 分 |
| KEY2 | PA14 | 选择 5 分钟 | 甲方 +2 分（短按）/ +3 分（长按 ≥600ms） |
| KEY3 | PA13 | 选择 15 分钟 | 乙方 +1 分 |
| KEY4 | PA12 | 选择 30 分钟 | 乙方 +2 分（短按）/ +3 分（长按 ≥600ms） |

4 个按键均采用外部上拉输入模式，按下为低电平。由于 PA12~PA15 默认复用 JTAG/SWD 调试功能，需要调用 `GPIO_PinRemapConfig()` 关闭复用。

### 3.4 蜂鸣器电路

- 驱动方式: TIM3_CH3 PWM 输出 (PB0)
- 频率: 1KHz (ARR = 999，50% 占空比)
- 控制: 鸣叫 50ms（计分）/ 100ms（选时间确认）后关闭

## 四、软件设计

### 4.1 主程序 (main.c)

主程序负责系统初始化、显示刷新调度和状态机按键处理:

```c
// 全局变量
volatile uint8_t  g_score_a, g_score_b;      // 两队分数
volatile uint8_t  g_game_quarter;            // 当前小节 (1-4)
volatile uint8_t  g_game_min, g_game_sec;    // 倒计时
volatile uint8_t  g_game_state;              // 状态机
volatile uint8_t  g_timer_running;           // 计时器使能
volatile uint8_t  g_refresh_flags;           // OLED 区域刷新标志
volatile uint8_t  g_quarter_minutes;        // 每节分钟数
volatile uint8_t  g_quarter_seconds;        // 每节秒数（demo 模式）
volatile uint32_t g_sys_tick;               // 1ms 全局计数器
```

**主循环核心逻辑**:

1. **OLED 刷新**: 检查 `g_refresh_flags` 位标志，按需刷新各区域（分区刷新避免整屏重绘）
2. **按键扫描**: 调用 `Key_Scan()` 获取按键编号
3. **状态机处理**:
   - IDLE: 根据 KEY1~4 设置时间，启动比赛
   - PLAYING: KEY1/3 加 1 分，KEY2/4 短按 +2 / 长按 +3
   - PAUSED: 任意键恢复计时
   - GAMEOVER: 任意键回到 IDLE

### 4.2 比赛逻辑 (Game.h + main.c)

**状态枚举**:

| 枚举值 | 含义 | 说明 |
|--------|------|------|
| `GAME_STATE_IDLE` | 等待选择时间 | 显示时间选择界面 |
| `GAME_STATE_PLAYING` | 比赛中 | 计时运行，可计分 |
| `GAME_STATE_PAUSED` | 暂停 | 节间休息，等待任意键继续 |
| `GAME_STATE_GAMEOVER` | 比赛结束 | 显示结果，任意键重置 |

**刷新标志位**:

| 标志 | 含义 |
|------|------|
| `RF_SCORE_A` | 甲方分数区 |
| `RF_SCORE_B` | 乙方分数区 |
| `RF_TIME` | 时间区 |
| `RF_STATUS` | 状态行 |
| `RF_OVER` | 比赛结束（全屏清） |
| `RF_TIMESEL` | 时间选择界面（全屏清） |
| `RF_ALL` | 重画所有区域 |

**核心函数**:

| 函数 | 功能 |
|------|------|
| `Game_SetQuarterTime(min, sec)` | 设置每节时间，支持正常分钟模式和 5 秒演示模式 |
| `Game_StartQuarter()` | 启动新小节，OLED 清屏，设置 `g_refresh_flags = RF_ALL` |
| `Game_AddScore(team, points)` | 给指定队伍加分（上限 199），触发蜂鸣器 50ms |
| `Game_Reset()` | 清零所有数据，`g_refresh_flags = RF_TIMESEL` 回到选时间界面 |

### 4.3 按键扫描 (Key.c)

**防连发机制**: `key_released` 标志。检测到低电平后锁定，抬手（高电平）才允许再次触发。

**消抖**: 检测到低电平后 `Delay_ms(20)` 二次确认，避免抖动误触发。

**长按检测**: 在 KEY2/KEY4 按下时，主循环通过 `Key_IsPressed()` 轮询 `g_sys_tick`，持续 ≥600ms 判定为长按。

```c
uint8_t Key_Scan(void)
{
    // 1. 检查 key_released: 为 0 时检测是否已抬手
    // 2. 检测到低电平 + key_released=1 → 消抖 20ms 二次确认
    // 3. 返回 KEY1_PRESS ~ KEY4_PRESS 或 KEY_NONE
}
```

### 4.4 OLED 显示 (OLED.c)

采用**软件模拟 I2C** 通信协议，通过 PB6/PB7 的 GPIO 推挽输出模拟 I2C 时序。

**显示布局**: 4 行 × 16 字符，每行占 2 页（128×16 像素）。

**分区刷新**: 仅在数据变化时更新对应区域，减少 OLED 写入量，提升响应速度。

### 4.5 蜂鸣器驱动 (Buzzer.c)

TIM3 配置为 PWM 模式:

| 参数 | 值 | 说明 |
|------|----|------|
| 定时器 | TIM3 | APB1 总线 |
| PSC | 72-1 | 72MHz / 72 = 1MHz |
| ARR | 999 | 1MHz / 1000 = 1KHz |
| 占空比 | 50% | 方波驱动 |
| 通道 | CH3 (PB0) | 复用推挽输出 |

`Buzzer_Beep(50)`: 设置 1KHz 频率 → 延时 50ms → 关闭输出。50ms 的时长既能提供清晰的按键反馈，又不会影响操作节奏。

### 4.6 比赛计时中断 (stm32f10x_it.c)

TIM2 配置为 1ms 中断:

| 参数 | 值 |
|------|----|
| PSC | 72-1 (72MHz / 72 = 1MHz) |
| ARR | 1000-1 (1MHz / 1000 = 1KHz) |
| 中断类型 | Update 事件 |

```c
void TIM2_IRQHandler(void)
{
    TIM_ClearITPendingBit(TIM2, TIM_IT_Update);
    g_sys_tick++;                                  // 供长按检测

    if (g_timer_running)
    {
        sec_cnt++;
        if (sec_cnt >= 1000)                      // 1 秒到
        {
            sec_cnt = 0;
            if (g_game_sec > 0)  g_game_sec--;
            else if (g_game_min > 0) { g_game_min--; g_game_sec = 59; }
            else {
                // 本节时间到
                g_timer_running = 0;
                if (g_game_quarter < 4) {
                    g_game_quarter++;
                    // demo 模式按秒重置，否则按分钟
                    g_game_state = PAUSED;
                } else {
                    g_game_state = GAMEOVER;
                }
            }
            g_refresh_flags |= RF_TIME;            // 触发时间区刷新
        }
    }
}
```

## 五、关键技术难点与解决方案

### 5.1 长按复用 +2/+3 分的设计

**难点**: 4 个物理按键需要支持 6 种计分操作。

**方案**: KEY2/KEY4 引入长按检测。短按 (<600ms) → +2 分，长按 (>=600ms) → +3 分。检测借助 `g_sys_tick` 全局时间戳，按下时记录 `start`，松开时计算差值。

### 5.2 按键防抖与防连发

**难点**: 机械按键存在抖动，且需要防止长按导致连续加分。

**方案**: 双重保护——软件消抖 (20ms 延时后二次确认) + `key_released` 标志（按下后锁定，抬手才允许再次触发）。

### 5.3 OLED 区域刷新优化

**难点**: 全屏刷新速度慢，影响按键响应体验。

**方案**: 引入 `g_refresh_flags` 位标志（RF_SCORE_A / RF_SCORE_B / RF_TIME / RF_STATUS 等），仅在数据变化时更新对应区域，时间选择和比赛结束界面使用 `RF_TIMESEL` / `RF_OVER` 触发整屏清屏重绘。

### 5.4 比赛计时精度

**难点**: 延时函数会阻塞主循环，不能用于精确计时。

**方案**: 使用 TIM2 硬件定时器的 1ms 中断独立计时。中断不影响主循环的按键处理和显示刷新，精度依赖 72MHz 晶振，误差 <0.001%。

### 5.5 OLED 软件 I2C 通信稳定性

**难点**: STM32F103C8T6 的硬件 I2C 存在已知 Bug（锁死问题）。

**方案**: 采用软件模拟 I2C，通过 GPIO 引脚模拟 SCL/SDA 时序，配合 SSD1306 数据手册的时序参数调整延时。

## 七、总结与展望

### 7.1 项目成果

本项目成功实现了一款基于 STM32F103C8T6 的篮球比赛电子记分牌，具备以下功能:

- OLED 实时显示两队比分、当前小节和倒计时
- 4 个按键控制计分: +1/+2/+3 分，含蜂鸣器提示音
- 用户可选每节时间（5 秒演示 / 5 分钟 / 15 分钟 / 30 分钟）
- 4 节精确倒计时，小节自动切换
- 比赛结束自动清屏显示 GAME OVER + 胜者，任意键回到选时间界面（完整轮询）

### 7.2 可扩展方向

1. **球员得分统计**: 增加子界面，记录每队 5 名球员个人得分
2. **24 秒进攻计时**: TIM4 独立计时器
3. **外部大屏**: UART/SPI 输出到 LED 点阵大屏
4. **蓝牙/WiFi 遥控**: 无线控制计分操作
5. **犯规与暂停计数**: 显示每队犯规次数与剩余暂停

### 7.3 个人收获

通过本项目，熟练掌握了 STM32 标准外设库的使用（GPIO、TIM、NVIC）、OLED 显示驱动开发、按键消抖算法、状态机设计以及前后台系统架构等嵌入式开发核心技能。
