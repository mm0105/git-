# 项目答辩稿 —— STM32G4 压力流量监控系统

> 一人答辩，时长 5~8 分钟。带 \* 的段落是可能被追问的要点。

---

## 一、开场（30 秒）

各位评委好，我做的项目是**基于 STM32G431 的压力流量监控系统**。系统读两路模拟/脉冲信号（压力、流量），按 MAN / AUTO 两种模式控制一路 PWM 输出，配套 LCD 三页菜单和 4 颗 LED 状态灯。下面我从功能、硬件、关键算法和可靠性设计四个方面介绍。

---

## 二、系统功能（1 分钟）

整个系统有 3 块功能：

1. **数据采集**：
   - 压力 P：从 R37 电位器经 PB15/ADC2_IN15 采集，按 0~3.3V → 0~10.0 BAR 线性映射
   - 流量 F：从 R39 经 TIM16 输入捕获测频率，按 1Hz = 0.005 L/M 换算
   - 累计流量 Q：F 每 100ms 积分一次，满 1L 才 +1

2. **控制输出**：
   - **PA1 (TIM2_CH2)** 输出 10kHz PWM（5%~95%），实际占空比 D_actual 每秒变化 1%
   - **PA7 (TIM17_CH1)** 同步输出 50Hz 舵机 PWM，D_actual 5-95 线性映射到 500-2500μs 脉冲
   - 手动模式：追 TarD
   - 自动模式：根据 P 与 TarP 偏差，决定 D 增/减/保持

3. **人机交互**：
   - LCD 三页：MAIN（监控）、OUTP（输出配置 TarD/TarP）、PARA（报警阈值 FH/FL/PL/DH）
   - 4 颗 LED：LD1 传感器异常 / LD2 堵塞 / LD3 泄漏 / LD4 动态调节
   - 4 颗按键：短按切页、加、减、模式/选项；长按 2 秒触发零点校准和累计清零

---

## 三、硬件接口（45 秒）

| 信号 | 引脚 | 资源 |
|---|---|---|
| PWM 输出（主） | PA1 | TIM2_CH2（10kHz 5%-95%） |
| PWM 输出（舵机） | PA7 | TIM17_CH1（50Hz 500-2500μs） |
| 压力模拟 | PB15 | ADC2_IN15 |
| 流量脉冲 | 板载 | TIM16_CH1 输入捕获 |
| LCD | FSMC | lcd.c 驱动 |
| 4 颗 LED | PC8~PC11 经 74HC595 | PD2 锁存 |
| 4 颗按键 | PB0/PB1/PB2/PA0 | 上拉输入 |

\* LED 用 74HC595 串行移位寄存器级联，只占 1 个锁存脚就扩出 8 路输出，节省 GPIO。

## 四、关键算法（2 分钟，重点）

### 4.1 压力换算 + 零点校准

```
P = (V - Voffset) / (3.3 - Voffset) × 10  BAR
```

Voffset 是零点偏置。长按 B3 2 秒 → 把当前电压写入 Voffset。这步的作用是消除传感器零漂和接线压降对零点的污染，**让"零压力时 P 一定等于 0"**。

\* 防止除零：if (range < 0.01) range = 0.01

### 4.2 PWM 1%/秒 平滑过渡（**评委必问**）

D_actual 是写到 CCR 的实际占空比。用户改 TarD/TarP 时，**不能一下跳过去**，要每秒 ±1% 追目标。

\* 关键设计：**时间基准统一用 HAL_GetTick() 的 1ms 计数**，不和 TIM3 实际频率挂钩。这样无论 TIM3 配成 1kHz 还是 10kHz，1%/秒都准。

### 4.3 MAN vs AUTO 算法

| 模式 | 控制目标 | 算法 |
|---|---|---|
| MAN | TarD（%占空比） | D_actual 1%/秒 追 TarD |
| AUTO | TarP（BAR 压力） | P<TarP-0.5 → D++; P>TarP+0.5 → D--; 死区保持 |

\* 死区 ±0.5 BAR 避免 D 在阈值附近反复抖动。

### 4.4 流量累计 Q

瞬时流量 F 单位是 L/M（升/分钟）。100ms 内增量为 `F × 0.1/60 L`。
为了避免 Q 每秒跳小数，用 Q_acc 累小数，**满 1L 才让 Q_value +1**。

### 4.5 LED 报警

- **LD1 传感器异常**：f > 8000Hz 立即亮（1s 内）
- **LD2 堵塞**：F < FL 且 D > DH **持续 2 秒** 才亮
- **LD3 泄漏**：F > FH 且 P < PL **持续 2 秒** 才亮
- **LD4 动态调节**：用户在 OUTP 改过目标且已生效 → 亮；目标达成 → 灭

\* LD2/LD3 加 2 秒防抖，避免瞬时尖刺误报。

### 4.6 长按 2 秒判定

\*（必问）"为什么有时候按 3 秒才触发？"
早期版本用"松手时算 dur"：必须先到 2s 再松手才触发，**任何 1.9s~2.0s 的抖动都会让 dur 落到短按区**，所以体感是"有时要按 3 秒才识别"。

新版本改成**按下到 2s 整立刻触发**，不等松手；用 `key_long_fired` 标志防重触。短按路径也补好：按下没到 2s 就松手才记短按。**0 抖动、0 漏触发**。

### 4.7 舵机 PWM 同步输出（PA7 / TIM17_CH1）

PA1 旁边再扩一路 PA7 (TIM17_CH1) 50Hz 舵机 PWM，**和 PA1 严格同步**：

| 定时器 | 引脚 | 频率 | 周期 | 计数时钟 | 用途 |
|---|---|---|---|---|---|
| TIM2_CH2 | PA1 | 10kHz | 100 | 1MHz | 比例阀 5%-95% |
| TIM17_CH1 | PA7 | 50Hz | 19999 | 1MHz | 舵机 0.5-2.5ms 脉冲 |

D_actual (5-95) 线性映射到脉冲宽度（500-2500μs）：

```
CCR = 500 + (D_actual - 5) × 200 / 9
D=5   → 500μs  → 0°
D=50  → 1500μs → 90°（中位）
D=95  → 2500μs → 180°
```

\* **关键**：`pwm_apply_duty()` 一次写两个 CCR（TIM2 + TIM17），所以**每一次 1%/秒 斜坡 PA1 和 PA7 同步变**，不会出现"PA1 动了 PA7 没动"或反之。

## 五、软件结构（1 分钟）

代码分 9 个模块：

```
app        主入口 / 主循环
key        按键状态机 + 长短按事件
lcd_ui     3 页 LCD + 100ms 数据刷新
adc_sensor ADC 采样 + 压力换算 + Voffset
pwm        TIM2 PWM 输出 + 1%/秒斜坡 + MAN/AUTO
flow       TIM16 测频 + 流量换算 + Q 累计
led        74HC595 驱动 + LD1~LD4 状态判定 + 2s 防抖
settings   全局状态变量（TarD / TarP / 阈值 / Voffset）
timer_task TIM3 中断里的周期任务分派（50/100/1000ms）
```

\* 模块化的好处：**改 ADC 不会动 PWM**，**改 LED 不会动按键**。答辩要举一个具体例子。

---

## 六、可靠性设计（45 秒）

1. **按键三态状态机**：空闲 → 按下确认（第二帧防抖） → 持续按下。机械抖动完全滤掉。
2. **长按立即触发**：2s 整立刻响应，不依赖松手时机。
3. **OUTP 页配置"待生效"**：按 B3/B4 只改预设值，**按 B1 退出时才把 tar_changed 置 1**，LD4 才亮。严格按规格"配置生效后"。
4. **LD2/LD3 2 秒去抖**：避免瞬时尖刺误报。
5. **Q 累计防溢出**：Q_value 限制 999999999L。
6. **PWM 上下限保护**：D_actual 钳到 [5, 95]。

## 七、答辩 Q&A 准备（可能被问的问题）

### Q1：长按 2 秒怎么实现？为什么有时需要 3 秒？
A：按键状态机读 GPIO 第二帧确认按下，**记录按下时刻的 HAL_GetTick()**。每帧检查"按下时长 ≥ 2000ms"则立即触发长按并置 key_long_fired 防重触。松手时若没触发过长按才记短按。
**3 秒才触发**的旧 bug：旧版本"松手时算 dur"，碰到按键抖动 dur 落在 1900~2100ms 跳变，可能 1.9s 就被误判为短按；用户再按到 3s 累计才到长按阈值。新版彻底解决。

### Q2：1%/秒怎么保证准？TIM3 改频率会变吗？
A：**时间基准用 HAL_GetTick() 的 1ms 计数**，不依赖 TIM3 实际频率。无论 TIM3 配 1kHz 还是 10kHz，pwm_ramp_tick 每次检查 `now - last_tick >= 1000ms` 触发，所以 1%/秒严格准。

### Q3：MAN 和 AUTO 区别？什么时候用哪个？
A：
- **MAN**：用户直接给定占空比（比如"出 30%"），D 1%/秒追到 30% 停。适合已知流量的场景。
- **AUTO**：用户给定目标压力，D 由 P-TarP 偏差自动调。适合维持压力的场景。
两种模式互斥，**用 B2 切**，LCD 监控页 `M=MAN/AUTO` 标识当前模式。

### Q4：零点校准有啥用？
A：消除传感器零漂和接线压降。R37 调到已知零位 → 长按 B3 → 当前电压存为 Voffset。之后 P 的换算就以这个 Voffset 为基准，**保证零压力时 P=0**。V 行显示的就是 Voffset，让你看到漂了多少。

### Q5：PWM 平滑过渡为什么不用中断定时？
A：用 HAL_GetTick() 而非 TIM3 计数的好处是**抗 TIM3 配置变更**。1ms tick 来自 SysTick，CPU 重启或 TIM3 重配都不影响 1%/秒 节拍。

### Q6：PA1 没接执行器怎么测？
A：本题台架 PA1 悬空，P 也不会被 D 改变。**这是正常的**——题目考的是**算法对不对**，不考物理闭环。
验证方法：手动旋 R37 改 P，看 D 1%/秒按规则动（追 TarD 或按 TarP 自动调），就对。

### Q7：累计流量 Q 为啥不是每秒显示？
A：F 单位是 L/M（升/分钟），每秒只走 1/60，**直接显示会有 6 位小数**。先累到 1L 才 +1，可读性好。

### Q8：LD2/LD3 2 秒去抖的目的？
A：瞬时尖刺（比如阀门瞬关、压力瞬变）会触发 F 异常，跳变一下立刻报警会很烦。**持续 2s 才报警**才稳。

### Q9：OUTP 页面改值，LD4 立刻亮还是退出才亮？
A：**退出时才亮**。按 B3/B4 只改预设值（B2 选 TarD/TarP），按 B1 切到监控页时如果 outp_dirty=1 才置 tar_changed=1 → LD4 亮。期间 LCD 实时显示预设值。

### Q10：PWM 输出具体接什么？
A：TIM2_CH2 → PA1，74HC595 那条线上没有。本题是算法考核，真闭环（PA1 接泵/电机）现场接上即可。

### Q11：PA7 又接什么？和 PA1 是什么关系？
A：**两路 PWM 共享 D_actual**。PA1 (TIM2_CH2) 是 10kHz / 5%-95% 比例阀输出；PA7 (TIM17_CH1) 是 50Hz 舵机输出，CCR 由 D_actual 线性映射到 500-2500μs 脉冲（D=5→0°、D=50→90°、D=95→180°）。`pwm_apply_duty()` 一次写两个寄存器，所以 1%/秒斜坡两路**严格同步**走。

### Q12：为什么 PA7 用 50Hz，PA1 用 10kHz？
A：**负载特性不同**。比例阀/电机是电流型负载，要高频开关才能线性控制（10kHz 远超人耳，听不到噪音），占空比=平均电压；舵机内部有位置反馈电路，**只认 50Hz 脉宽位置信号**（周期 20ms、脉宽 0.5-2.5ms 对应 0-180°），给 10kHz 反而会被伺服 IC 当成噪声忽略。

### Q13：PA1 和 PA7 的 D_actual 同步精度怎么样？
A：**一个变量、两次赋值、原子操作**。`pwm_apply_duty()` 里就两行 `__HAL_TIM_SetCompare()`，中间没有中断打断风险（CCR 写入是 32 位原子操作，CCR 预装载影子寄存器在下一个更新事件才生效）。视觉上看不到两路有先后。

---

## 八、收尾（15 秒）

总结一下：系统用 9 个模块化文件实现了**采集 + 控制 + 报警 + 交互**四大块，重点是**1%/秒 PWM 平滑过渡**和**2 秒长按即时触发**两个细节。**双路 PWM 同步**（PA1 比例阀 + PA7 舵机共享 D_actual）演示时用示波器两路同时看，能直观看到 1%/秒同步变化。代码已经过充分测试，长按、零点校准、累计清零、模式切换都稳定。**演示结束，请评委提问。**
